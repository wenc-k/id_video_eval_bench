# ID Video Evaluation System

A unified evaluation toolkit for **ID-preserving video generation**.  
The input of each test case is a **reference image (or reference image folder)**, a **text prompt**, and a **generated video**. The toolkit evaluates three high-level dimensions:

- **ID Consistency**
- **Text Consistency**
- **Video Quality**

The evaluation script reads a JSON file of test cases and produces three output JSON files:

1. **case-level results** for all samples
2. **id-level results** aggregated over all cases belonging to the same identity
3. **overall summary results** aggregated over the full benchmark

## 3. Input Format

The evaluation input is a JSON file containing a **list of test cases**.  
Each case must include the following fields:

- `id_index`
- `ref_img_path`
- `video_path`
- `prompt`

Where:

- `ref_img_path` is the path to the reference image
- `video_path` is the path to the generated video to be evaluated
- `prompt` is the text prompt used to generate the video

---

## 4. Evaluation Workflow

The evaluation is divided into **two stages**.

### Stage 1

Run Stage 1 in the `opens2v` environment:

```bash
conda activate opens2v
bash eval_stage1.sh
```

Stage 1 computes the following metrics:

* `aes_score`
* `cur_score`
* `arc_score`
* `gme_score`
* `motion_fb`
* `motion_lk`
* `naturalscore_1`
* `naturalscore`

It also creates the intermediate directory:

* `_v2_work/input_videos`

Stage 1 outputs the following files:

* `case_results_v2.json`
* `id_results_v2.json`
* `summary_results_v2.json`

### Stage 2

Run Stage 2 in the `opens2v_qalign` environment:

```bash
conda activate opens2v_qalign
bash eval_stage2.sh
```

Stage 2 will:

* reuse the `input_videos` generated by Stage 1
* compute `motion_smoothness`
* merge it with the existing Stage 1 metrics
* compute the final Human-Domain total score

### Important

Stage 1 and Stage 2 must use the **same input JSON file**, and the case order must remain unchanged.

---

## 5. Output Files

### Stage 1 Outputs

Stage 1 writes the following files to `output_dir`:

* `case_results_v2.json`
* `id_results_v2.json`
* `summary_results_v2.json`

#### `case_results_v2.json`

Contains one result entry for each input case.
The original input fields are preserved, and Stage 1 metric values are appended.

#### `id_results_v2.json`

Contains one entry per `id_index`, obtained by averaging all cases belonging to the same identity.

#### `summary_results_v2.json`

Contains the dataset-level average of all Stage 1 metrics.

### Stage 2 Outputs

Stage 2 writes the following files to `output_dir`:

* `case_results_with_qalign_ms.json`
* `id_results_with_qalign_ms.json`
* `summary_human_domain_total.json`

#### `case_results_with_qalign_ms.json`

Contains the Stage 1 case results with additional fields:

* `motion_smoothness`
* normalized metric values
* `total_score`

#### `id_results_with_qalign_ms.json`

Contains identity-level averaged results, including `total_score`.

#### `summary_human_domain_total.json`

Contains the final dataset-level summary, including:

* averaged raw metrics
* averaged normalized metrics
* final `total_score`

## 6. Human-Domain Merge Weights

The final total score is computed using the **Human-Domain** weighting scheme, without NexusScore.

| Metric            | Source field                          | Weight |
| ----------------- | ------------------------------------- | -----: |
| Aesthetic         | `aes_score`                           |   0.18 |
| Motion Smoothness | `motion_smoothness`                   |   0.09 |
| Motion Amplitude  | `motion_fb`                           |   0.03 |
| Face Similarity   | `cur_score`                           |   0.25 |
| Text Consistency  | `gme_score`                           |   0.15 |
| Natural Score     | `naturalscore` / `naturalscore_1`     |   0.30 |
